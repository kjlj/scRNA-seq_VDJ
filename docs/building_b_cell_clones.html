Building Ig clones

2023-08-25

Overview

For the T cells the clonotype labels have already been created by joining the V, J and CDR3AA sequence.

This workflow uses the IMGT CDR3 definition which excludes the 104-Cys and 118-Phe/Tyr. There are other CDR3 definitions that incorporate these residues. The equivalent ‘CDR3’ can be obtained fro, the junction and junction_aa columns reported by IgBLAST which will include the anchor residues.

As the B cell chains undergo somatic hypermutation (SHM) need to use a method for defining clonal groups that permits grouping of similar but not exact sequences. Hierarchical clustering of the Ig sequences CDR3 sequence is one approach used to identify groups of cells that carry sequences that are derived from the same original progenitor B cell. These cells will carry heavy and light chains with similar nucleotide sequences that have diverged due to the accumulation of point mutations.

Clustering must be undertaken for each individual as a clonal lineage can only exist within a single individual. Similar Igs from different individuals are ‘convergent’ antibodies, that may have arisen from response to the same antigen, rather than ‘divergent’ members of the same clone. The same type of clustering can be used to explore convergence (also known as public clones) in antibody responses but this is in addition to building clonal lineages within an individual’s immune response.

In the example datasets the pbmc and tumour samples are from different individuals so clones will be built separately. If 10x datasets includes multiple samples that require de-multiplexing via hashing antibodies or SNPs that the B cell V(D)J data must be subset before building clones.

An excellent resource for post-processing of 10x data including building clonal relationships can be found at https://immcantation.readthedocs.io/en/stable/tutorials/10x_tutorial.html. This approach can be undertaken in R, but can also export data and use alternative tools such as cd-hit but this requires some additional coding to wrangle the cd-hit output into a format that can be joined to the Cell Ranger and IgBLAST cell barcode level information. There is some information available here.

The RScript for this workflow available on github in the workshop respository.

Here, we build upon the Ig data generated from the combining IgBLAST and cellranger workflow.

Set the clustering threshold

Before performing the hierarchical clustering the clustering threshold needs to be defined. This is done by comparing the pairwise hamming distance between sequences in a dataset to find each sequences nearest distance. This should result in a bimodal distribution that defines ‘clones’ (sequences very near to each other) and ‘unrelated’ (sequences more distant from each other).

For this, we can use the IGH data.

The SHazaM package expects IgBLAST column names in their orignal format so we need to extract those columns and rename them to remove the _iglbast_igh suffix that we appended previously:

## the shazam package uses the IgBLAST columns, so can just select those and rename appropriately
pbmc.igh.data <- pbmc.data %>% 
  select(barcode, sequence_id = contig_id_igh, ends_with("_igblast_igh")) %>%
  rename_with(~ str_replace(.x, "_igblast_igh", ""), everything()) %>%
  filter(!(is.na(cdr3))) #need to remove any unpaired entries as they will cause errors

tumour.igh.data <- tumour.data %>%

Plotting the distances to look for the valley between the bimodal distribution that should provide a threshold between ‘clonal’ and ‘unrelated’ sequences:

#plotting the distribution
pbmc.dist.p1 <- ggplot(pbmc.igh.data %>% filter(!is.na(dist_nearest)),
             aes(x = dist_nearest)) +
  theme_bw() +
  xlab("Hamming distance") + ylab("Count") +
  scale_x_continuous(breaks = seq(0, 1, 0.1)) +
  geom_histogram(color = "white", binwidth = 0.02) +
  scale_y_continuous(expand = expansion(mult = c(0, 0.05))) +
  theme_custom()

There don’t appear to be many clonally related sequences in the PBMC dataset, but a threshold of 0.1 should split the ‘clones’ from the ‘unrelated’ sequences:

#add a line for threshold

  geom_vline(xintercept = 0.1, linetype = "dashed", colour = "firebrick")

The clones will be generated using the hierarchicalClones function from the SCOPer package. This function will add a ‘clone_id’ column to the tibble and entries that share the same ‘clone_id’ are considered members of the same clonal lineage:

#add the clone id columns for PBMC
pbmc.input <- hierarchicalClones(pbmc.input, 
                              threshold = threshold,
                              cell_id = "barcode",
                              method = "nt",
                              linkage = "average",
                              junction = "cdr3",
                              only_heavy = FALSE, 
                              split_light = TRUE,
                              summarize_clones = FALSE)

As PBMC and tumour data are in same tibble, the clone_ids are not unique as the same ‘clone_ID’ can be in both datasets. To avoid possible issues during analysis, create a unique clone ID by combining the ‘dataset’ and ‘clone_ID’ values together in a new label:

#add a unique labels for the clns that is the clone_id + dataset
bcr.cln.data <- bcr.cln.data %>%
  mutate(unq_clone_id = ifelse(is.na(clone_id), NA_character_, paste(dataset, clone_id, sep = "_")))

Finally, create a quick summary of clone sizes to check if there are any expanded clones:

#quick summary of clone sizes
cln.size.summary <- bcr.cln.data %>%
  filter(!(is.na(unq_clone_id))) %>% #remove cells that lack a clone call
  group_by(dataset, unq_clone_id) %>% #group by clone_id & dataset
  summarise(cln_size = length(barcode)) %>% #count the number of barcodes in each clones
  ungroup() %>% #remove the grouping
  arrange(desc(cln_size)) #reorder the data from largest to smallest clone size across the two datasets
